A project exploring the effect on pay on investment banks' performance

Introduction
One of the most fundamental questions any employer must ask is how much to pay an employee. Careers in the City are no exception, despite been renowned for its large salaries. Banks further raised these salaries (c.30%) last year to cope with burnout and the war for talent. But does this provide value for money? This project aims to establish a link between pay for bankers and their performance, measured by the fees the bank brings in each year. My initial expectation is that of a negative correlation: lower-performing banks to need to pay a higher salary to attract talent, where better performing banks can leverage their prestige.

Unfortunately, banks do not actively publish their annual fees and employee salaries. Furthermore, data providers, such as Refinitiv, which publish APIs containing this information, charge thousands each year for access. This project's primary aim is therefore to use free resources to provide this data and perform some basic analysis. Please note: at no point does this project attempt to establish causality – merely tries to establish a link.

The chart above shows the total compensation (base + bonus pay). Section 4: Regressing bank performance on compensation

This section regresses the charts in section 3. We cannot regress all ranks together, because one cannot compare Analyst 1 pay to VP 3 pay in the same regression. With the scraper auto-saving backups, each year we would get more datapoints improving the reliability of our regression.

Ideally, this chart would be made using a Vega trellis, but I could not get the trellis to run all the regressions individually. The data in the charts can be replicated here.

As expected, there is a strong correlation between total compensation and bank performance. This is because bank bonuses tend to be a function of the performance of the bank. However, the relationship is striking, with an R2>0.4 in half the regressions.

Unsurprisingly, the R2 is much lower when considering just base pay. However, there is still an upward trend, suggesting that there is some correlation between base pay and performance. Contrary to my initial expectation: banks which pay more have, on average, better performance.

Conclusion

Overall, many of the project's aims have been met. We have established a positive correlation between pay and performance. Furthermore, data (in a space which is highly protected by paywalls) has been uncovered, collection automated, and used in analysis. The greatest weakness of the project is that there are simply not enough datapoints for completely robust regressions. However, the data collection is (almost) fully automated, allowing charts to be refreshed each year. Future work should use this data to perform more robust regressions, with more datapoints to improve accuracy.
